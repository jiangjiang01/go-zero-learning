一、背景与痛点

原工作任务

作为前端开发，需要完善后端技术栈，学习 Go 语言和 go-zero 框架，目标是掌握完整的后端开发能力，能够独立开发包含用户管理、权限控制、商品管理、订单管理等功能的电商系统。

痛点普遍性与高价值

这是所有程序员都面临的普遍问题：
  • 技术迭代快：新框架、新语言、新工具层出不穷，不跟进容易被淘汰
  • 学习成本高：传统学习方式效率低，从零到掌握新技术需要 3-6 个月
  • 成功率低：据统计，70%+ 的程序员在新技术学习过程中半途而废
  • 职业发展瓶颈：技术栈单一限制职业发展，跨栈学习困难重重
  • 时间成本巨大：每天投入 2-3 小时，持续数月，但效果不佳

高价值体现：解决这个问题不仅能提升个人学习效率，更能帮助整个程序员群体突破学习瓶颈，实现技术栈快速扩展，提升职业竞争力。

耗时情况（量化数据）

传统学习方式的时间成本：
  • 啃官方文档：预计需要 2-3 个月（每天 2-3 小时），但进度缓慢，经常学到后面忘记前面
  • 跟视频教程：每天学习 2-3 小时，持续 1 个月后陷入困境，多次产生放弃念头
  • 环境适配调试：每次遇到问题需 1-2 天调试，累计浪费 10-15 天
  • 总体评估：从零基础到能够独立完成完整项目，预计需要 3-6 个月，且成功率 < 30%
  • 总时间成本：180-360 小时（按每天 2 小时计算，持续 3-6 个月）

核心难点（痛点分析）

  • 学习效率低：官方文档过于详细，花大量时间学习短期内用不上的知识点，有效学习时间仅占 30-40%
  • 实践脱节：文档和教程中的示例与实际工作场景严重脱节，看完还是不知道如何落地
  • 环境适配问题：教程使用的 Go 版本与本地环境存在差异，完全复刻代码也频繁报错，每次调试耗时 1-2 天
  • 调试困难：面对层出不穷的报错信息，反复调试却不得要领，在试错与返工的循环中消耗大量时间
  • 理解不深入：机械复制代码，换个场景就懵，根本没理解背后逻辑，学习深度不足，无法举一反三
  • 学习节奏不可控：视频教程节奏紧凑，稍一分神就难以跟上，学习体验差，容易产生挫败感

二、AI解决方案

方案创新性：从工具到教练的转变

核心创新点：
  • 角色创新：将 AI 从"代码生成工具"转变为"专属学习教练"，系统性引导学习而非简单代码输出
  • 方法创新：首创"先给 low 写法，再优化"的教学策略，通过"踩坑"加深理解，而非直接给最优解
  • 流程创新：设计完整的四步学习工作流，将"盲目试错"变成"精准落地"
  • Prompt 工程创新：通过精心设计的 Prompt 让 AI 遵守教学原则，实现个性化、系统化的学习引导

使用的AI工具

Cursor：AI 代码助手，作为"专属学习教练"，负责代码生成、问题解答、原理讲解
通用AI助手：用于项目搜索和初始文档生成，提供学习路径规划

核心方法论：AI 四步学习法（完整工作流）

  1. 找到想学习技术的现有项目
    操作：使用 Prompt 让 AI 搜索 GitHub 上高质量参考项目（star > 1000）
    输出：获得高质量学习参考项目列表
  2. 让 AI 输出详细的初始文档
    操作：使用 Prompt 让 AI 生成包含项目结构、核心教学原则、学习计划的完整文档
    输出：获得可直接使用的学习初始文档（README.md）
  3. 创建空目录，让 Cursor 开始引导
    操作：将文档交给 Cursor，让它根据教学原则一步步引导实现功能
    输出：在 AI 引导下逐步实现完整项目
  4. 交给时间，持续学习
    操作：按照"先给 low 写法，再优化"的方式，通过"踩坑"加深理解
    输出：掌握技术栈，具备独立开发能力

Prompt 工程：精心设计的 Prompt 模板

Prompt 设计思路：
  • 明确目标：清晰定义期望的输出结果
  • 结构化要求：使用编号列表，确保输出结构化、可执行
  • 约束条件：设置明确的筛选标准（如 star > 1000）
  • 上下文传递：在后续 Prompt 中引用前一步的输出，形成连贯的工作流

Prompt 1：项目搜索（第一步）

帮我找 GitHub 上使用 go-zero 框架的完整项目，要求： 1. star 数量超过 1000 2. 项目结构清晰，有完整的业务逻辑 3. 包含前后端完整实现 4. 有详细的文档说明 请列出项目名称、GitHub 链接、技术栈、项目特点。

Prompt 2：初始文档生成（第二步）

我找到了学习参考项目：[项目名称和链接] 请帮我生成一份详细的学习初始文档，包含： 1. 项目基本信息（项目名称、技术栈、学习目标） 2. 核心教学原则（AI 必须遵守的规则，包括角色定位、代码输出策略等） 3. 学习流程示例 4. 项目结构说明 5. 分阶段的学习计划 这个文档后续会直接给 Cursor，让它从空目录开始带我一步步实现。 请确保文档结构清晰，可以直接复制使用。

Prompt 3：开始学习（第三步）

我在学习 go-zero 项目，请先查看 README.md 了解当前进度。 当前需要实现 [具体功能]。 请按照 README.md 中的教学原则： 1. 分小步输出，每次只输出一小步 2. 先给出能运行但有问题的版本（low 写法） 3. 让我先遇到问题，然后解释问题原因 4. 最后给出优化方案，解释为什么更好

核心教学原则（AI 必须遵守的规则）

角色定位：AI 作为资深开发，负责引导和教学；用户作为学习者，跟着 AI 一步步实现功能

代码输出策略（三步教学法）：
  • 步骤1：分小步输出：每次只输出一小步，逐步实现，避免一次性给出完整代码
  • 步骤2：先给"low 的写法"：先给出能运行但有问题的版本，让学习者先遇到问题，看到警告/错误
  • 步骤3：暴露问题 → 解释问题 → 给出优化方案：说明为什么会有警告/错误，解释这种写法的风险和问题，然后提供更好的实现方式

工作流优势：通过这种"踩坑"式学习，学习者不仅能学会"怎么做"，更能理解"为什么这么做"，实现深度理解而非机械复制。

可操作性：方法易于学习和复制

快速上手（5 分钟开始学习）：
  1. 复制 Prompt 1，让 AI 搜索参考项目（2 分钟）
  2. 复制 Prompt 2，生成初始文档（2 分钟）
  3. 创建空目录，将文档粘贴到 README.md（1 分钟）
  4. 使用 Prompt 3，开始学习（立即开始）

适用场景：学习任何编程语言、框架、工具，只需替换技术栈名称即可复用整个工作流。
